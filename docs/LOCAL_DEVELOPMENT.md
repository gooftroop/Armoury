# Local Development with LocalStack

LocalStack emulates AWS services locally, enabling full end-to-end development and testing without a remote AWS account.

## Prerequisites

- **Node.js** >= 24.0.0 (see `.nvmrc`)
- **npm** (ships with Node)
- **Docker Desktop** (or Docker Engine + Docker Compose v2)
- **jq** â€” `brew install jq` / `apt install jq`
- **awslocal** â€” `pip install awscli-local` or `brew install awscli-local`
- **zip** â€” usually pre-installed

## Quick Start

```bash
npm run localstack:up          # Start LocalStack container
npm run localstack:bootstrap   # Build, deploy Lambdas, provision API Gateways, generate .env.local
npm run localstack:down        # Tear down (removes volumes)
```

## What `localstack:bootstrap` Does

1. Creates a shared IAM execution role for all Lambdas
2. Builds all services in parallel via Turborepo (`npx turbo run build --filter='./src/services/*'`)
3. For each service with `hasLambda: true` in `infra/localstack/services.json`:
    - Zips the `dist/` output
    - Creates or updates the Lambda function in LocalStack
    - If the service has a `restApiId`: creates a REST API Gateway with a `{proxy+}` catch-all route and ANY method
4. Generates `.env.local` at the repo root with `*_BASE_URL` env vars pointing to the LocalStack API Gateway endpoints
5. Prints a summary of all API Gateway URLs

## Environment Variables

### Generated by bootstrap

Running `npm run localstack:bootstrap` generates a `.env.local` file at the repo root with API Gateway URLs:

```
CAMPAIGNS_BASE_URL=http://localhost:4566/_aws/execute-api/campaigns-api/local
USERS_BASE_URL=http://localhost:4566/_aws/execute-api/users-api/local
MATCHES_BASE_URL=http://localhost:4566/_aws/execute-api/matches-api/local
FRIENDS_BASE_URL=http://localhost:4566/_aws/execute-api/friends-api/local
```

Each client library in `src/shared/clients/*/src/config.ts` reads its corresponding env var with a localhost fallback:

```typescript
// src/shared/clients/campaigns/src/config.ts
export const CAMPAIGNS_BASE_URL = process.env.CAMPAIGNS_BASE_URL ?? 'http://localhost:3000';
```

When `.env.local` is present, the clients automatically connect to the LocalStack API Gateways instead of the default localhost ports.

**How apps load `.env.local`:**

- **Next.js (web)**: Loads `.env.local` from the project root automatically. The `*_BASE_URL` vars are available via `process.env` server-side without extra configuration.
- **Expo (mobile)**: Does not auto-load `.env.local`. Set the env vars in your shell before starting the dev server, or copy the values into `src/mobile/.env`.

### Required in `.env` (repo root)

- `LOCALSTACK_AUTH_TOKEN` â€” Required for LocalStack Pro features. Can be empty for the free tier.

### Injected into each Lambda

- `IS_LOCAL=true`
- `AWS_ENDPOINT_URL=http://localhost:4566`
- `SECRET_NAME=armoury/{service}/config`

## Adding a New Service

1. Add an entry to `infra/localstack/services.json` with `name`, `secretPath`, `restApiId`, `handler`, `dbPort`, and `hasLambda`
2. Re-run `npm run localstack:bootstrap`
3. The new Lambda, API Gateway, and `.env.local` entry are created automatically

## Architecture

`infra/localstack/docker-compose.yml` starts the LocalStack container with an init hook. On startup, `infra/localstack/seed.sh` runs inside the container and seeds Secrets Manager with dummy config for each service.

`infra/localstack/bootstrap.sh` runs from the host. It builds, packages, and deploys Lambdas and API Gateways. `infra/localstack/services.json` is the single source of truth for all service metadata.

## Troubleshooting

| Problem                           | Solution                                                                      |
| --------------------------------- | ----------------------------------------------------------------------------- |
| `awslocal: command not found`     | `pip install awscli-local` or `brew install awscli-local`                     |
| `jq: command not found`           | `brew install jq` or `apt install jq`                                         |
| Bootstrap fails on "zip"          | `apt install zip` or `brew install zip`                                       |
| Lambda invocation returns 502     | Check Lambda logs: `awslocal logs tail /aws/lambda/{name} --region us-east-1` |
| API Gateway returns 403           | Ensure bootstrap completed and the API was deployed to the `local` stage      |
| "Function already exists" warning | Safe to ignore. Bootstrap is idempotent (creates or updates).                 |
| Container won't start             | Check Docker is running; verify port 4566 is free: `lsof -i :4566`            |
| `.env.local` not updated          | Re-run `npm run localstack:bootstrap`. It overwrites `.env.local` each time.  |
